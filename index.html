06 — KG HIERARCHY
Hierarchy & Graph Construction
C3 — Weighted Louvain

Three-Layer Architecture — สีแสดง layer level ต่างกัน

L3 — Global Layer
TOP · CORPUS-LEVEL

สรุปจากหลาย communities ผ่าน map-reduce synthesis เพื่อตอบคำถามระดับทั้งคอร์ปัสได้

L2 — Community Layer
MID · THEME CLUSTERS

จัดกลุ่ม entities เป็นธีม เช่น economy, weapons balance, anti-cheat ผ่าน Louvain algorithm

L1 — Entity Layer
BASE · ATOMIC TRIPLES

ชั้นล่างสุด เก็บ entities และ typed triples แบบตรงไปตรงมา รากฐานของทุกอย่าง

C3 — Weighted Louvain

Weighted Louvain
Contribution C3

Louvain ที่ edge มีน้ำหนักต่างกันตาม predicate type ใน 8-Predicate Schema แทนที่จะเท่ากันหมด

Louvain Algorithm
C3 Base

อัลกอริทึมแบ่ง community ที่นิยม เพราะเร็วและมีแนวคิด modularity รองรับ

Predicate-to-Weight

Community Detection
Graph Algorithm

Modularity
Quality Metric

Semantic Coherence

Map-Reduce Synthesis

07 — QUERY ROUTING
Tri-Path Adaptive Query Router
C4 — 3 Routes

Query Route Types — แต่ละสี = destination layer ต่างกัน

LOCAL
→ L1 ENTITY LAYER

คำถามเฉพาะ entity โดยตรง ดึง L1 subgraph เพื่อตอบ ความมั่นใจสูง ชัดเจน

"Wallbang คืออะไร?"
HYBRID
→ L2 THEN DRILL L1

คำถามกึ่งเฉพาะกึ่งภาพรวม ต้องดู community ก่อนแล้วเจาะลงไป L1

"อาวุธอะไรโดนวิจารณ์เรื่องสมดุลมากที่สุด?"
GLOBAL
→ L3 MAP-REDUCE

คำถามภาพรวมทั้งคอร์ปัส ใช้ L3 map-reduce synthesis เพื่อตอบ

"ผู้เล่นบ่นเรื่องอะไรใน CS2 มากที่สุด?"

C4 — Router Mechanics

Tri-Path Adaptive Router
Contribution C4

ตัว routing 3 ทางของงานนี้ ดูว่าคำถามควรใช้ข้อมูลจากชั้นไหน LOCAL / HYBRID / GLOBAL

Few-Shot Classifier
Router Mechanism

ตัวจำแนกที่ดูตัวอย่างก่อนตอบ ใช้ exemplars ช่วยตัดสิน query type

Nine-Exemplar Prompt
Router Prompt

prompt มีตัวอย่าง 9 ข้อ — 3 LOCAL, 3 GLOBAL, 3 borderline

Confidence Score
Router Output

คะแนนความมั่นใจ ถ้าต่ำเกิน threshold ส่งไป HYBRID

c ≥ 0.8 → route confirmed
Rule-Based Fallback
Safety Net

ถ้าพบ "overall", "most players" → บังคับ GLOBAL

Top-N Evidence
Retrieval

ดึง L1 triples 5–10 รายการที่เกี่ยวข้องมากที่สุดแนบคำตอบ
